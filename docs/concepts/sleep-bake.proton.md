# Sleep Bake 概念定義

## 目的
Sleep Bakeは、覚醒中に増加した短期差分を整理・統合し、次周期の応答品質と監査可能性を維持するための必須プロセスである。

## 定義
- Sleepは、未統合差分の整理工程を指す。
- Bakeは、整理結果を安定記憶へ統合し、監査イベントとして固定する工程を指す。
- Sleep Bakeは演出機能ではなく、性能維持と責任追跡のための運用手続きである。

## 背景
- 覚醒継続により文脈差分が蓄積すると、参照コストが上昇する。
- 文脈窓制約に近づくと応答品質が低下し、判断の一貫性が崩れる。
- この現象を疲労状態として扱い、睡眠処理で回復させる。

## 効果
1. 応答性能の回復。
2. 記憶階層の再編。
3. 監査証跡の固定。

## 境界
- Sleep Bakeは人格規範を自動改変する仕組みではない。
- 運用者の承認方針なしに人格核を書き換えない。

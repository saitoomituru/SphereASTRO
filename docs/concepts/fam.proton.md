# FAM（Fold Access Mapper）概念定義

## 目的
FAM（Fold Access Mapper）は、対話システム内の思考経路を可視化し、責任境界を維持するための構造モデルである。

## 定義
- FAMは、入力・解釈・判断・出力を段階化して記述するための抽象レイヤである。
- FAMは、推論エンジンの挙動を直接説明するものではなく、人格モデルがどの前提で判断したかを記録する枠組みである。
- FAMは、GUI状態管理やAPI仕様とは分離して扱う。

## 構成要素
1. 観測（Observation）: ユーザー入力・環境状態・履歴を収集する。
2. 文脈化（Contextualization）: 収集情報を価値基準と責務境界に照合する。
3. 判断（Decision）: 人格モデルの規範に基づき方針を確定する。
4. 表現（Expression）: 判断結果を対話出力へ変換する。
5. 監査（Audit）: 入力根拠と判断痕跡を追跡可能に保持する。

## 非目的
- FAMは「AIの意識」を証明する理論ではない。
- FAMはモデル固有プロンプトの隠蔽層ではない。
- FAMはMCP（User / Assistant / System）を内部モデルとして採用しない。MCPは外部通信上の便宜的プロトコルであり、責任記述の基盤とはしない。

## 設計原則
- 人格的判断と演算資源を分離する。
- どの判断がどの根拠で行われたかを記録可能とする。
- ベンダー固有APIに依存しない。

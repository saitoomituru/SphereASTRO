# EdoHAGE 監査チェーン概念定義

## 目的
EdoHAGEは、対話と記憶統合に関する因果関係を改ざん検知可能な形で保持する監査チェーンである。

## 定義
- 監査イベントを時系列で連結するハッシュチェーン構造を採用する。
- 各イベントは前イベントのハッシュを保持し、追記専用（append-only）で管理する。
- ローカル台帳を基本とし、必要時に外部監査基盤へ橋渡し可能とする。

## 記録対象
- 入力素材ハッシュ
- 人格状態ハッシュ（運用タグ・設定識別子を含む）
- 実行環境ハッシュ
- 依代ポリシーハッシュ
- 統合差分ハッシュ
- 実行時刻・イベントID

## 保証範囲
- 完全な改ざん防止ではなく、改ざん検知と責任追跡を保証する。
- 監査チェーン単体で倫理判断の正当性を保証しない。
- 監査チェーンは再現性確保のための証跡層である。
- 事故時はAI概念全体を対象化せず、社会・司法で問題性が認定された当該ファイル単位で監査・隔離・検証できることを保証する。

## 運用原則
1. 監査イベントは削除しない。
2. 再生成で同一入力なら同一正規化結果を得る。
3. 監査失敗時は統合処理を完了扱いにしない。
4. 押収・証拠保全時は、荒御魂ファイルと監査チェーンの対応関係を保持する。

# GUI責務定義

## 原則
GUIは状態管理と可視化のみを担当する。思考・判断・責任判定は担当しない。

## GUIの責務
- 入力フォーム、会話履歴、設定画面の表示。
- ユーザー操作をイベントとして発火。
- ViewModelが返す状態を描画。
- エラー状態を利用者に明示。

## GUIの非責務
- 応答内容の意味判断。
- 人格規範の評価。
- 依代選択の自動最適化。
- 監査チェーンの整合判定。

## 実装ガイド（SwiftUI）
1. Viewは`@State`と`@Binding`を用いた表示制御に限定する。
2. 非同期処理開始はViewModel経由で行う。
3. 画面遷移条件は状態列挙体で明示する。
4. 監査関連の表示は「結果表示」のみに留め、計算は下位層へ委譲する。

## 期待効果
- UI改修時に判断ロジックが破壊されない。
- 実装者が責任境界を誤解しない。
- テスト設計をUIテストとロジックテストに分離できる。
